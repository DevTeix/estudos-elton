1. Fundamentos do Protocolo TCP/IP: Além da Teoria

Objetivo Técnico: Dominar a inspeção de tráfego e diagnosticar problemas de conectividade/segurança em profundidade.

Ação Ferramentas Passo a Passo Simplificado Ganho Técnico
Análise de Conexão TCP `Wireshark` / `tcpdump` 1. Inicie a captura. 2. Abra uma conexão (ex: `telnet 80`). 3. Filtre por `tcp.flags.syn == 1`. 4. **Identifique o 3-Way Handshake:** SYN, SYN-ACK, ACK. Saber se a falha é na rede, no serviço ou no firewall. Essencial para defesa (identificar varreduras de porta).
Diagnóstico de Rotas (Segurança) `tracert` / `traceroute` 1. Execute `tracert google.com`. 2. Analise os "saltos" (hops). 3. **Segurança:** O último salto antes de um serviço crítico deve ser um Firewall/IPS. Se a rota for estranha, pode haver sequestro de rota. Entender o caminho do pacote e identificar pontos vulneráveis.
Configuração de Firewall (Camada 3/4) Windows Firewall Avançado / `iptables` **Exemplo: Bloquear Ping (ICMP):** Windows: Nova Regra -> Personalizada -> Protocolo ICMPv4 -> Bloquear. Linux:
sudo iptables -A INPUT -p icmp -j DROP
 Domínio prático em regras L3/L4, base de qualquer Firewall de Borda.

2. Posicionamento e Fundamentos de Soluções de Segurança

Objetivo Técnico: Saber como configurar a "Regra Ouro" de cada solução e garantir a comunicação entre elas.

2.1. Firewall (Filtro de Tráfego - Borda/Interno)

Solução Implementação Prática (Simples) Aumento de Nível Técnico
Firewall de Borda (Rede) **Regra Ouro:** Criar uma regra **"Deny All" (Bloquear Tudo)** no final da lista. Depois, adicionar a regra **"Permitir Acesso à Internet"** (Portas 80, 443, 53) apenas para a rede interna. Entender o conceito de **Implicit Deny** (Negação Implícita), a base de todo firewall moderno.
Host Firewall (Windows/Linux) **No Windows, via GPO:** Criar uma GPO para bloquear tráfego de saída (Outbound) em portas incomuns (ex: 6667, 1234) que malware costuma usar. Usar o conhecimento de GPO para **gerenciar a segurança do endpoint de forma centralizada**, simulando uma defesa interna.

2.2. EDR, Antivírus e Controle de Aplicação

Solução Implementação Prática (Simples) Aumento de Nível Técnico
Controle de Aplicação (Via GPO) **Windows:** Usar **Software Restriction Policies (SRP)** ou **AppLocker** via GPO para: 1. Definir uma regra que permita a execução de arquivos apenas do diretório `C:\Program Files` e `C:\Windows`. 2. Bloquear arquivos executáveis (`.exe`, `.ps1`) de rodar no diretório `C:\Users\*`. Mapear o controle granular de execução. Essa é uma das principais táticas de defesa contra *ransomware* e *malware fileless*.
Host IPS (Linux - SELinux) **Linux:** Configurar o SELinux no modo `Enforcing` (ativo) e depois usar o comando `semanage` para restringir o acesso de um serviço específico (ex: um servidor web) a um diretório que ele não precisa acessar. Dominar a segurança robusta do Linux no nível de kernel, essencial para servidores críticos.

2.3. MFA e SIEM

Solução Implementação Prática (Simples) Aumento de Nível Técnico
MFA (Conceito + AD) **Raciocínio:** O MFA deve ser aplicado onde o risco é maior. **Ex:** Usuários do grupo "Domain Admins" (AD) devem ter o MFA forçado. Ligar a segurança de autenticação ao **Active Directory**, entendendo que a gestão de identidade é a primeira linha de defesa.
SIEM (Coleta Centralizada) **Windows Event Forwarding:** Configurar um servidor Windows como *Coletor* de Eventos e as estações de trabalho como *Fontes* de Eventos, focando em logs de **Security** e **Application**. Entender como os dados chegam ao SIEM, que é 80% do trabalho de um analista de segurança.

3. Frameworks: Da Estratégia à Detecção

Objetivo Técnico: Usar a lógica do ataque (Kill Chain) para criar regras de SIEM e mapear defesas (MITRE).

3.1. Cyber Kill Chain (Caminho do Ataque)

Fase da Kill Chain Ação do Atacante (Exemplo Ransomware) Onde a Defesa Deve AGIR (Sua Tarefa)
1. ReconhecimentoEscanear portas abertas na Luniv.**Ação:** Firewall de Borda deve bloquear/limitar escaneamentos externos.
2. ArmamentoCriar um payload de ransomware.**Ação:** Nenhuma. Ocorre fora da rede.
3. EntregaEnviar o payload por e-mail (Phishing) ou USB.**Ação:** AntiSpam/Filtro de Conteúdo (e-mail) e Controle de Aplicação (USB).
4. ExploraçãoO usuário executa o anexo, explorando vulnerabilidades (CVE).**Ação:** EDR/Antivírus deve detectar a assinatura ou o comportamento de exploração.
5. InstalaçãoO ransomware se copia e cria chaves no Registro.**Ação:** EDR/Controle de Aplicação (via GPO) deve bloquear a criação de chaves incomuns.
6. C2 (Comando e Controle)O ransomware se comunica com o servidor do bandido.**Ação:** Firewall ou IPS de Borda deve bloquear conexões de saída (Outbound) para IPs maliciosos conhecidos.
7. Ações no ObjetivoCriptografar todos os arquivos.**Ação:** Backup offline (última defesa) e EDR/Controle de Aplicação para interromper o processo de criptografia.

3.2. Framework MITRE ATT&CK: Mapeando Defesas

Tática (O Quê) Técnica (Como) Comando de Detecção (No SIEM/Logs) Solução de Defesa Principal
Persistence (T1548) Bypass UAC (T1548.002) Monitorar criação de novas entradas no Registro do Windows em chaves de execução automática (`Run`, `RunOnce`). EDR e GPO (para restringir alterações).
Execution (T1059) PowerShell (T1059.001) Coletar e analisar todos os logs de execução do PowerShell. Buscar por scripts codificados (`-EncodedCommand`). EDR/Antivírus (para bloquear) e SIEM (para alertar a execução).
Defense Evasion (T1036) File Deletion (T1070.004) Monitorar logs do sistema de arquivos para exclusão massiva de logs de segurança ou arquivos de shadow copy (`vssadmin`). SIEM (para alerta) e EDR (para reversão).
